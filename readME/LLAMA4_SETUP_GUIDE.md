# Llama 4 Scout Setup Guide

This guide walks you through downloading and configuring **Llama 4 Scout** using the official `llama-stack` CLI tool.

## Step 1: Install llama-stack

```bash
pip install llama-stack
```

Or update if you already have it:

```bash
pip install llama-stack -U
```

## Step 2: List Available Models

See what models are available:

```bash
llama model list
```

For all versions including older ones:

```bash
llama model list --show-all
```

## Step 3: Download Llama 4 Scout

### For Instruction Following (Recommended)

```bash
llama model download --source meta --model-id Llama-4-Scout-17B-16E-Instruct
```

### For Base Model (Pretrained)

```bash
llama model download --source meta --model-id Llama-4-Scout-17B-16E
```

### For Protections Model

```bash
llama model download --source meta --model-id Llama-Guard-4-12B
```

## Step 5: Configure Your System

### Option A: Auto-Detection (Recommended)

The system will automatically detect models downloaded via llama-stack in these locations:
- `~/.llama/models/`
- `~/.cache/llama/models/`
- `./models/llama/`

### Option B: Manual Configuration

If the model is in a different location, set the environment variable:

**Windows (PowerShell):**
```powershell
$env:LLAMA_STACK_MODEL_PATH="C:\Users\YourName\.llama\models\Llama-4-Scout-17B-16E-Instruct"
$env:LOAD_LLAMA_TRANSFORMERS="true"
```

**Windows (CMD):**
```cmd
set LLAMA_STACK_MODEL_PATH=C:\Users\YourName\.llama\models\Llama-4-Scout-17B-16E-Instruct
set LOAD_LLAMA_TRANSFORMERS=true
```

**Linux/Mac:**
```bash
export LLAMA_STACK_MODEL_PATH=~/.llama/models/Llama-4-Scout-17B-16E-Instruct
export LOAD_LLAMA_TRANSFORMERS=true
```

## Step 6: Verify Installation

Run your Python code to check if the model is detected:

```python
from enrichment_module import SemanticNodeEnricher

# This will print detection messages
enricher = SemanticNodeEnricher(
    use_llama=True,
    prefer_local=True
)
```

You should see:
```
Llama 4 model loaded via transformers from llama-stack: [path]
Llama enricher initialized with backend: transformers
```

## Step 7: Use in Your Pipeline

```python
from enrichment_module import SemanticNodeEnricher
from semantic_node_enhanced import SemanticNode

# Initialize with Llama 4
enricher = SemanticNodeEnricher(
    use_llama=True,        # Enable Llama 4
    use_gemini=True,       # Keep Gemini as fallback
    prefer_local=True      # Use local Llama before cloud Gemini
)

# Enrich a node
node = SemanticNode(name="max_V", value="0.133", unit="m/s")
enricher.enrich_node(node)

print(node.definition)  # Generated by Llama 4 locally!
```

## Troubleshooting

### Model Not Found
- Check the download completed: `llama model list`
- Verify the path: Check `~/.llama/models/` or your custom path
- Set `LLAMA_STACK_MODEL_PATH` explicitly

### Out of Memory
- The 17B model requires significant RAM (40GB+)
- Consider using a smaller model or quantization
- Use `llama-cpp-python` with quantized GGUF models instead

### Download Failed
- Check your internet connection
- Verify the custom URL is still valid (48-hour window)
- Try downloading again (up to 5 attempts allowed)
- Request a new URL if needed

### Transformers Not Loading
- Ensure `LOAD_LLAMA_TRANSFORMERS=true` is set
- Install required packages: `pip install transformers torch`
- Check available disk space (models are large)
